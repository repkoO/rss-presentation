TypeScript: Supercharge Your JavaScript

by Kirill Medvedev

The Problem with JavaScript

  • Dynamic Typing
    Errors at runtime, not compile time
  • Weak Type System
    Hard to refactor, maintain
  • Growing Pains
    Codebases become unmanageable

TypeScript to the Rescue: Static Typing

  • Catch Errors Early
    During development, not production
  • Better Readability
    Clear intentions, self-documenting
  • IDE Superpowers
    Autocompletion, refactoring, navigation

TypeScript Basics: Types and Interfaces

  • Interfaces
    Contracts for objects
  • Type Aliases
    Custom type names
  • Primitive Types
    string, number, boolean, any

Practical Example: Function Type Annotations


// JavaScript
function greet(name) { 
 return "Hello " + name;  
}

// TypeScript
function greet(name: string): string {  
 return "Hello " + name;  
}

Benefits: Clear signatures, fewer runtime errors

Working with Classes in TypeScript

  • Encapsulation
    Bundling data with methods
  • Inheritance
    Parent-child relationships
  • Access Modifiers
    public, private, protected
  • Polymorphism
    Multiple forms, one interface

Generics: Reusable Components

  • Define Type Parameters
    Use <T> as placeholder
  • Apply to Functions/Classes
    function identity<T>(arg: T): T
  • Reap Benefits
    Type safety with flexibility

Union and Intersection Types


// Union Types
type StringOrNumber = string | number;  // Accept multiple type options (like logical OR)

// Intersection Types
type Employee = Person & Worker;  // Combine multiple types (like logical AND)

  • Conditional Types
    Types that depend on other types
  • Utility Types
